Create Unity Catalog HTTP Connection

Let's create a Unity Catalog HTTP connection to connect to an external MCP server. For this example, we'll set up a connection to a GitHub MCP server.

‚ö†Ô∏è

Prerequisites

You need CREATE CONNECTION privileges on the Unity Catalog metastore to create HTTP connections.

Get Your GitHub Personal Access Token

Before creating the connection, you'll need a GitHub Personal Access Token (PAT) to authenticate with the GitHub API.

1. Go to Developer Settings ‚Üí Personal Access Tokens

2. Create a new token (fine-grained or classic)

3. Grant the necessary permissions (repo, issues, etc.)

Create the HTTP Connection

In your Databricks workspace, navigate to Catalog ‚Üí External Data ‚Üí Connections and click on + Create connection and select HTTP. Enter the following details:

  • ‚Ä¢ Connection name: github_mcp_connection_(name_prefix)
  • ‚Ä¢ Connection type: HTTP
  • ‚Ä¢ Auth type: Bearer token
  • ‚Ä¢ Click Next
  • ‚Ä¢ Host: https://api.githubcopilot.com
  • ‚Ä¢ Port: 443
  • ‚Ä¢ Bearer token: _insert_token_here_
  • ‚Ä¢ Click Create
  • ‚Ä¢ Base path: /mcp
  • ‚Ä¢ Check "Is MCP connection" (set to True)
  • ‚Ä¢ Click Create connection
[Alternative] Create GitHub MCP Connection programmatically using SQL (SQL Editor or Notebook)sql
-- Create HTTP connection for GitHub MCP server
-- Replace {name_prefix} with your unique identifier (e.g., your username)
-- Replace <insert_token_here> with your GitHub Personal Access Token
CREATE CONNECTION github_mcp_connection_{name_prefix}
  TYPE HTTP
  OPTIONS (
    host 'https://api.githubcopilot.com',
    port '443',
    base_path '/mcp',
    is_mcp_connection 'true',
    bearer_token '<insert_token_here>'
  );

Important Configuration Notes

  • ‚Ä¢Replace {name_prefix} with your unique identifier to avoid naming conflicts (e.g., your username)
  • ‚Ä¢Replace <insert_token_here> with your actual GitHub PAT
  • ‚Ä¢Make sure to set is_mcp_connection to 'true' to enable MCP functionality
